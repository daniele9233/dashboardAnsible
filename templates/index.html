Vault

Decrypt and inspect Ansible vault files. Read-only.

Required to decrypt vault.yml, vault_cavalid.yml, vault_storage.yml, vault_pgadmin.yml and vault_grafana.yml.

vault.yml
group_vars/all/vault.yml

Master Ansible vault containing cluster secrets and SSH keys.

vault_cavalid.yml
group_vars/all/vault_cavalid.yml

CA-valid certificate bundle for the trusted Rancher deployment.

vault_storage.yml
group_vars/all/vault_storage.yml

Azure Managed Disk identifiers (subscription, RG, disk name) for the static PV role.

vault_pgadmin.yml
group_vars/all/vault_pgadmin.yml

pgAdmin admin email and password for the web GUI (PostgreSQL external manager).

vault_grafana.yml
group_vars/all/vault_grafana.yml

Grafana admin user and password for the observability dashboard.

Dry-Run

Run the playbook in --check --diff mode. Nothing is applied to the cluster.

Dry-Run · Full stack
via maestra ansible-playbook site.yml --check --diff

Simulate the full stack install in one shot: pre-check + Traefik + Rancher CA-Valid + Static PV (Azure Disk) + pgAdmin 4. Single vault prompt, single playbook, order enforced by site.yml.

tag: rancher_ca_valid
CA-Valid mode

Simulate Rancher install with corporate CA certificates (trusted chain).

site-self-signed.yml
Self-Signed mode

Simulate Rancher install with its own self-signed cert (cert-manager). Separate playbook from site.yml.

tag: aks_static_pv
Static PV (Azure Disk)

Simulate binding of an existing Azure Managed Disk as PersistentVolume + PVC.

tag: pgadmin
pgAdmin 4 (monitoring)

Simulate pgAdmin install with a dedicated PVC (managed-csi, 4Gi) and Traefik Ingress.

tag: grafana
Grafana 11 (monitoring)

Simulate Grafana install with a dedicated PVC (managed-csi, 5Gi) and Traefik Ingress. Shares the monitoring namespace with pgAdmin.

Dry-run reports changes that would be applied. Use this to validate vars and tags before a real deploy.

Deploy

Apply the Rancher playbook to the live AKS cluster. Changes are real.

This will install / upgrade Rancher on the production cluster.
Deploy · Full stack
via maestra

Install the full stack in sequence: Traefik → Rancher CA-Valid → Static PV (Azure Disk) → pgAdmin 4. Single vault prompt, single playbook. Excludes Rancher Self-Signed (mutually exclusive with CA-Valid).

Deploy · CA-Valid

Install Rancher with corporate CA-signed certificates.

Deploy · Self-Signed

Install Rancher with its bundled self-signed certificate (cert-manager). Runs site-self-signed.yml — separate playbook from the full stack.

Deploy · Static PV (Azure Disk)

Bind an existing Azure Managed Disk as PersistentVolume + PVC. Reclaim policy: Retain.

Deploy · pgAdmin 4

Install pgAdmin in namespace monitoring. Dedicated PVC (managed-csi, 4Gi) and Traefik Ingress.

Deploy · Grafana 11

Install Grafana in namespace monitoring. Dedicated PVC (managed-csi, 5Gi) and Traefik Ingress. Co-tenant of pgAdmin.

Uninstall

Remove Rancher from the AKS cluster. Destructive action — confirmation required.

This will run bash uninstall-rancher.sh

All Rancher resources, namespaces and CRDs will be removed. Workloads outside the Rancher namespace are not touched. No vault password required.

Cancel
This will run bash uninstall-pgadmin.sh

Removes pgAdmin (Deployment, Service, Ingress, Secrets, PVC pvc-pgadmin) from namespace monitoring. The namespace itself is deleted only if empty. The underlying Azure Disk (managed-csi, reclaim Delete) is automatically removed by AKS. No vault password required.

Cancel
This will run bash uninstall-grafana.sh

Removes Grafana (Deployment, Service, Ingress, Secrets, PVC pvc-grafana) from namespace monitoring. The namespace is preserved if pgAdmin or other workloads are still present. The underlying Azure Disk (managed-csi, reclaim Delete) is automatically removed by AKS. No vault password required.

Cancel
Delete PersistentVolumes

List the PVs currently registered in the cluster and delete the ones you select. Bound PVs are still attached to an active PVC: deleting them is risky. The underlying Azure Disk is removed only if the PV's reclaimPolicy is Delete; with Retain the disk remains in Azure and must be removed manually from the portal/CLI.

Click Refresh list to load the PersistentVolumes.

SSH Commands

Tre flussi di accesso: Rancher (CA Valid / Self-Signed) e Dashboard Ansible. Esegui i comandi nell'ordine indicato.

Rancher · CA Valid

Accesso diretto via DNS aziendale + Traefik (cert wildcard AlmavivA, lucchetto verde).

cert valido
1. Apri il browser su Windows
https://rancher.app.iac-svil.almaviva.it/dashboard/home
Accesso diretto · nessun tunnel necessario

Il DNS aziendale risolve rancher.app.iac-svil.almaviva.it sull'Internal LoadBalancer di Traefik (10.207.201.138), raggiungibile dalla rete aziendale via VNet/ExpressRoute. NIENTE SSH tunnel, NIENTE kubectl port-forward, NIENTE riga nel file hosts. Se in passato avevi aggiunto 127.0.0.1 rancher.app.iac-svil.almaviva.it al file hosts, RIMUOVILA.

Rancher · Self-Signed

Accesso diretto al servizio Rancher (TLS gestito da dynamiclistener).

self-signed
1. SSH tunnel · Windows PowerShell (lascia aperto)
ssh -i "C:\Users\D.Pascolini\Desktop\IAC-Azure\ALMDEPIAC01LTAZ_key 1.pem" -L 8443:localhost:8443 azureuser@10.207.201.136
2. kubectl port-forward · dentro la sessione SSH della VM
kubectl port-forward -n cattle-system svc/rancher 8443:443
3. Browser su Windows
https://localhost:8443/login
Avviso certificato

Il browser mostrerà un warning di certificato non valido (atteso in modalità self-signed). Clicca Avanzate → Procedi comunque a localhost.

Dashboard Ansible

Avvia e accedi a questa stessa dashboard da Windows.

controller
1. Avvia il demone sulla VM (se non gia' attivo)
sudo systemctl start aks-dashboard.service
2. SSH tunnel · Windows PowerShell (lascia aperto)
ssh -i "C:\Users\D.Pascolini\Desktop\IAC-Azure\ALMDEPIAC01LTAZ_key 1.pem" -L 8080:localhost:8080 azureuser@10.207.201.136
3. Browser su Windows
http://localhost:8080/#/dashboard
Demone systemd · aks-dashboard.service

La dashboard parte automaticamente al boot della VM e si riavvia in caso di crash. Per fermarla: sudo systemctl stop aks-dashboard. Per i log live: journalctl -u aks-dashboard -f. Setup completo in HOW-TO, sezione 10.

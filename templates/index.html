Dry-Run

Esegue il playbook in modalità --check --diff. Nessuna modifica viene applicata agli host.

Vuoto = tutti gli host dell'inventory. Limita l'esecuzione a un singolo gruppo.

Dry-Run · playbook completo
--check --diff

Simula l'esecuzione del playbook del progetto attivo. Mostra i cambiamenti che verrebbero applicati, senza toccare gli host.

Usa il dry-run per validare variabili e inventory prima di un deploy reale. Alcuni task con when dinamici possono comportarsi diversamente in check mode.
Vedi i file del playbook

Deploy

Applica il playbook del progetto attivo agli host reali. Le modifiche sono reali.

Questa azione esegue il playbook contro gli host di produzione.
Deploy · playbook completo
ansible-playbook

Esegue il playbook del progetto attivo applicando le modifiche agli host selezionati.

Controlli

Verifiche rapide e non distruttive sul progetto attivo: syntax-check, lista task/host e ping di connettività.

--syntax-check
Syntax check

Verifica che il playbook sia sintatticamente valido, senza eseguirlo.

-m ping
Ping host

Controlla la raggiungibilità SSH degli host dell'inventory.

--list-tasks
Lista task

Elenca i task che verrebbero eseguiti dal playbook.

--list-hosts
Lista host

Mostra su quali host verrebbe eseguito il playbook.
